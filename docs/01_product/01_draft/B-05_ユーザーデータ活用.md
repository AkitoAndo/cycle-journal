# B-05: 過去ユーザーデータの活用設計

## 概要

ユーザーのジャーナルや対話履歴を活用し、より自然で一貫した支援を行えるようにする。ユーザーの成長を追跡し、感情や価値観の変化を把握しながら、個別化された問いかけや提案を実現する。

過去データは「**土**（環境・記憶）」として扱い、未来へつなげる根拠とする。

---

## 活用方針

### 1. 文脈保持

- 過去の会話やジャーナルを「土（Soil）」として蓄積し、現在の対話に反映する
- **例**: 「前に書いていた〇〇は、いまどんな形で残っている？」

### 2. 感情変化の追跡

- B-01の感情ラベルを時系列で保存し、感情の揺れや安定性を確認する
- **例**: 「以前は不安と書いていたけど、最近は落ち着きが増えてきたようだね。」

### 3. 価値観の安定と変化

- B-02の価値観ラベルを軸に、どの価値観が一貫しているか、どの価値観が変化しているかを把握する
- **例**: 「根っこにある"自由"はずっと変わらないね。」

### 4. 習慣・行動の記録

- B-04の行動提案や実施結果をログ化し、水（継続）の流れとして記録する
- **例**: 「小さな習慣を続けている様子が見えるよ。流れが安定してきているね。」

### 5. 気づきの蓄積

- 空（Sky）の気づきや直感的な言葉をピックアップし、未来の方向性を映す資源とする
- **例**: 「直感的に"挑戦したい"と言っていたことが、だんだん形になってきているね。」

---

## 実装ルール

| ルール | 説明 |
|--------|------|
| 個人特定情報の非保存 | 個人特定可能な情報は保存しない |
| 抽象化 | ラベルや要約を使い、抽象化されたデータで記録する |
| 成長支援目的のみ | 利用はユーザーの成長支援のためであり、広告や商用利用は行わない |
| 同意前提 | 履歴の参照はユーザーの同意を前提とする |

```json

  "journal_summary": "今日は安心感があった",

  "values": ["自己肯定"],
  "actions": ["朝のランニングを継続"],
  "insights": ["新しい挑戦をしたい気持ち"]
}
```

---

## Cycleモデルとの対応

| Cycle要素 | データでの役割 |
|----------|---------------|
| 土（Soil） | 蓄積されたジャーナルや記憶 → データ保存の基盤 |
| 水（Water） | 行動や習慣の継続ログ → 時系列の記録 |
| 根（Root） | 価値観の記録 → 一貫性と変化を把握 |
| 幹（Trunk） | 意志や姿勢の表明 → ユーザーの言葉に反映 |
| 枝（Branch） | 思考の広がり → 過去に検討した選択肢の記録 |
| 葉（Leaf） | 日常行動の具体例 → 行動ログ |
| 実（Fruit） | 成果や喜びの記録 → 達成体験 |
| 空（Sky） | 気づきや直感の記録 → 未来方向性の示唆 |

---

## まとめ

B-05は過去ユーザーデータを「土」として蓄積し、成長の流れを見える化する設計。AIコーチはこの記録を参照しながら、自然な問いかけや提案を行い、ユーザーが自らの歩みを確認できるように支援する。
