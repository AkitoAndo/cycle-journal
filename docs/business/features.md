# 機能一覧

Cycle Journal を構成する 11 の主要機能です。各機能は Cycle モデルに基づいてユーザーの内省と成長を支援します。

## F1: 会話エンジン

AIコーチとの対話を管理する中核機能。

| 項目 | 内容 |
|------|------|
| 概要 | Cycle モデルに基づいた共感的コーチング対話 |
| LLM | AWS Bedrock / Claude 3 Haiku |
| 応答スタイル | 1〜3文、非指示型、オープンエンドな問い |
| 入力 | ユーザーメッセージ + 任意の日記コンテキスト |
| 出力 | コーチ応答 + メタデータ（感情・Cycle要素） |

**コーチの7つのルール:**
1. 言葉をジャッジせずに受け止める
2. 穏やかで開かれた問いを投げかける
3. ユーザーの言葉を鏡のように返す
4. 答えを与えず、自己発見を促す
5. 1〜3文の簡潔な応答
6. 意図的な「間」を大切にする
7. 過去の言葉・行動と結びつける

## F2: 感情分析

ユーザーの発話から感情を検出・ラベリングする機能。

| 項目 | 内容 |
|------|------|
| 入力 | ユーザーの自然言語テキスト |
| 出力 | 感情ラベル（例: fatigue, joy, anxiety） |
| 活用先 | セッション要約、コーチ応答の調整 |
| 実装 | LLM のレスポンスメタデータとして返却 |

## F3: 価値観分析

対話の蓄積からユーザーの深層的な価値観パターンを抽出する機能。

| 項目 | 内容 |
|------|------|
| 概要 | Cycle モデルの「根（Root）」に対応 |
| 入力 | 複数セッションの対話履歴 |
| 出力 | 価値観パターン・信念の傾向 |
| ステータス | 設計済み・未実装 |

## F4: 質問生成

コーチが投げかける問いを文脈に応じて動的に生成する機能。

| 項目 | 内容 |
|------|------|
| 概要 | ユーザーの状態・文脈に最適な問いを選択 |
| 手法 | システムプロンプト + Cycle 要素に基づく生成 |
| パターン | 傾聴 → 問いかけ → ミラーリング → 深掘り |

## F5: 状態判定

ユーザーの現在の心理状態を Cycle モデル上で判定する機能。

| 項目 | 内容 |
|------|------|
| 概要 | ユーザーが Cycle のどの段階にいるか判定 |
| 要素 | 土・水・根・幹・枝・葉・実・空 |
| 出力 | `cycle_element` としてメタデータに含める |
| 活用 | コーチの応答トーンや問いの方向性を調整 |

## F6: 行動提案

対話の中から具体的な行動タスクを提案する機能。

| 項目 | 内容 |
|------|------|
| 概要 | Cycle モデルの「葉（Leaf）」に対応 |
| 入力 | コーチとの対話内容 |
| 出力 | ActionTask（タイトル・説明・期限） |
| 特徴 | ユーザーの言葉から自然に提案 |

## F7: ふりかえり支援

タスク完了後の4ステップ構造化振り返りを支援する機能。

| 項目 | 内容 |
|------|------|
| 概要 | Cycle モデルの「実（Fruit）」に対応 |
| ステップ | 事実 → 感情 → 学び → 次のアクション |
| UI | ウィザード形式（プログレスバー付き） |
| 出力 | TaskReflection レコード |

**4ステップの詳細:**

| ステップ | 問い | Cycle 要素 |
|----------|------|-----------|
| 1. 事実の確認 | 何をした？ | 葉（Leaf） |
| 2. 感情の観察 | どう感じた？ | 根（Root） |
| 3. 学びの抽出 | 何に気づいた？ | 実（Fruit） |
| 4. 次への調整 | 次はどうする？ | 枝（Branch） |

## F8: 文脈管理

対話セッション間の文脈を保持・活用する機能。

| 項目 | 内容 |
|------|------|
| 概要 | 過去の対話・振り返りを新しいセッションに引き継ぐ |
| スコープ | セッション内文脈 + クロスセッション文脈 |
| 日記連携 | 日記内容をコーチとの対話コンテキストに注入可能 |
| ステータス | 基本実装済み（セッション内）、高度な文脈管理は未実装 |

## F9: ユーザーデータ管理

ユーザーデータの保存・管理を行う機能。

| 項目 | 内容 |
|------|------|
| ローカルデータ | 日記エントリ（UserDefaults） |
| クラウドデータ | コーチセッション・タスク（API経由） |
| 認証情報 | Apple ID トークン（Keychain） |
| エクスポート | データエクスポート機能あり |

**データ保存方針:**

| データ種別 | 保存先 | 理由 |
|-----------|--------|------|
| 日記 | ローカル（UserDefaults） | プライバシー最優先 |
| コーチセッション | クラウド（API） | デバイス間同期のため |
| タスク・振り返り | クラウド（API） | デバイス間同期のため |
| 認証トークン | Keychain | セキュリティ確保 |
| ユーザー設定 | ローカル | オフライン対応 |

## F10: 安全フィルター

ユーザーの安全を守るためのコンテンツフィルタリング機能。

| 項目 | 内容 |
|------|------|
| 概要 | 危険な発言や状態を検出し適切に対応 |
| 対象 | 自傷・自殺関連、極度のストレス、危機的状態 |
| 対応 | 専門機関への案内、エスカレーション |
| ステータス | 設計済み・基本実装は LLM プロンプトに含む |

## F11: エラーハンドリング

システム全体のエラーを適切に処理する機能。

| 項目 | 内容 |
|------|------|
| API エラー | 標準エラーレスポンス形式（code, message, details） |
| ネットワーク | リトライ・フォールバック応答 |
| LLM エラー | モック応答によるフォールバック |
| クライアント | エラー表示 + リトライ UI |
