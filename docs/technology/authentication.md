# 認証

## 概要

Cycle Journal は **Sign in with Apple** を認証方式として採用しています。Apple の OAuth 2.0 フローを利用し、JWT（JSON Web Token）ベースでトークンを検証します。

## 認証フロー

```
┌───────────┐      ┌──────────────┐      ┌───────────────┐
│  iOS App  │      │ Apple Server │      │ Cycle Journal │
│           │      │              │      │     API       │
└─────┬─────┘      └──────┬───────┘      └───────┬───────┘
      │                    │                      │
      │ 1. Sign in with    │                      │
      │    Apple ボタン    │                      │
      │ ──────────────────>│                      │
      │                    │                      │
      │ 2. identityToken   │                      │
      │    (JWT) 返却     │                      │
      │ <──────────────────│                      │
      │                    │                      │
      │ 3. POST /auth/verify                      │
      │    {identity_token: "eyJ..."}             │
      │ ─────────────────────────────────────────>│
      │                    │                      │
      │                    │ 4. Apple 公開鍵取得   │
      │                    │ <────────────────────│
      │                    │                      │
      │                    │ 5. 公開鍵返却        │
      │                    │ ────────────────────>│
      │                    │                      │
      │                    │              6. JWT 検証
      │                    │              ・署名検証 (RS256)
      │                    │              ・aud 検証

      │                    │                      │
```

## トークン検証の詳細

### Apple 公開鍵

| 項目 | 値 |
|------|-----|
| エンドポイント | `https://appleid.apple.com/auth/keys` |
| 形式 | JWK（JSON Web Key） |
| キャッシュ TTL | 1 時間 |
| アルゴリズム | RS256 |

### JWT クレーム検証

| クレーム | 検証内容 |
|---------|---------|
| `iss` | `https://appleid.apple.com` であること |
| `aud` | Bundle ID（`akito-ando.CycleJournal`）であること |
| `exp` | 有効期限内であること |
| `sub` | Apple ユーザー ID として使用 |
| `email` | ユーザーメールアドレスとして使用 |

## クライアント側の実装

### トークン管理

| 操作 | 保存先 | 説明 |
|------|--------|------|
| 保存 | Keychain | identityToken を安全に保存 |
| 取得 | Keychain | API リクエスト時に Bearer ヘッダーに付与 |
| 更新 | Apple SDK | アプリ起動時にトークンの有効性を確認 |
| 削除 | Keychain | サインアウト時にクリーンアップ |

### API リクエスト時のヘッダー

```
Authorization: Bearer {identityToken}
```

## Lambda Authorizer

API Gateway の Lambda Authorizer として実装済みですが、**トークンリフレッシュの仕組みが未整備のため、現在は無効化**されています。

| 項目 | 値 |
|------|-----|
| タイプ | トークンベース |
| キャッシュ TTL | 5 分 |
| ステータス | 実装済み・無効化中 |

### 有効化に必要な対応
- トークンリフレッシュメカニズムの実装
- CDK の `api_stack.py` でコメントアウトを解除

## セキュリティ考慮事項

- Apple のリレーメールアドレスによりユーザーの実メールを匿名化可能
- 公開鍵はキャッシュされるが、1時間で更新される
- Keychain はデバイスの暗号化チップで保護される
- HTTPS 通信により転送中のデータは暗号化される
